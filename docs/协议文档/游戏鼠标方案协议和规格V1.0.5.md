2: 无线 VID（0x1A86） PID（0x8300），驱动预留接口，可修改

**数据包格式：**

**PC - \> 设备**

|      |      |           |         |     |                                          |
|:----:|:----:|:---------:|:-------:|:---:|:----------------------------------------:|
| BYTE |  0   |     1     |    2    |  3  |                  4~ 63                   |
| 描述 | 0xBA | 0X8A/0XFF | SUB_CMD | LEN | 与命令相关的数据内容,整包长度不足64的补0 |

**0xBA:** PC -\> 设备的包头

**0xA5/0xFF**: 0x8A表示无线方式通过DONGLE的RF过来的命令

> 0XFF表示是有线USB方式来的命令

**SUB_CMD:** 对应命令集中的一个

**LEN:** Byte4开始，有效的数据数量

**设备 - \> PC**

|      |      |           |         |     |                                          |
|:----:|:----:|:---------:|:-------:|:---:|:----------------------------------------:|
| BYTE |  0   |     1     |    2    |  3  |                  4~ 63                   |
| 描述 | 0x5A | 0X8A/0XFF | SUB_CMD | LEN | 与命令相关的数据内容,整包长度不足64的补0 |
| 描述 | 0x5B | 0X8A/0XFF | SUB_CMD |  1  |                 Err Code                 |

**0x5B:** 设备 -\> PC的包头

**0xA5/0xFF**: 0x8A表示是无线方式通过DONGLE的RF过来的命令

> 0XFF表示是有线USB方式来的命令

**SUB_CMD:** 对应命令集中的一个

**LEN:** Byte4开始，有效的数据数量

**0x55:** 设备 -\> PC,通知PC指令异常

**Err Code： 详见附录 Error Code Definitions**

**重传机制: 由PC端检测到ACK异常（包括比对不一致）时，最多重传
3次，重传间隔100ms**

|  0x01:键盘组合键   | Modify key |    Key code    | 2<sup>nd</sup> code |
|     0x02:鼠标      |    0x00    |   Button ID    |        0x00         |
|   0x03:多媒体键    |    0x00    | 多媒体键值高位 |   多媒体键值低位    |

| 0x06:功能按键 暂无 |    0x00    |  Function ID   |        0x00         |
|    0x07:宏定义     |    0x00    |    Macro ID    |        0x00         |
|     0x08:滚轮      |    0x00    |      方向      |  连续滚动（0x00）   |

**a) 当Byte1为0x00时，表示该按键的功能为键盘单键**

Byte2: 0x00

Byte3: 按键键码（请参见附录的HID Usage ID）

Byte4: 0x00

例如：

若一个按键设为单键A

|        |        |        |        |
|:------:|:------:|:------:|:------:|
| Byte 1 | Byte 2 | Byte 3 | Byte 4 |
|  0x00  |  0x00  |  0x04  |  0x00  |

若一个按键设为单键Shift

|        |        |        |        |
|:------:|:------:|:------:|:------:|
| Byte 1 | Byte 2 | Byte 3 | Byte 4 |
|  0x00  |  0x00  |  0xE1  |  0x00  |

注意：

1、假设将按键A设为单键Ctrl，那么当用户按下A+C时，其效果应该与按下Ctrl+C一样。

**b) 当Byte1为0x01时，表示该按键的功能为键盘组合键**

Byte2: Modify key，指明组合键中包含那些modify key。

bit0=1，表示L-Shfit键

bit1=1，表示L-Ctrl键

bit2=1，表示L-Alt键

bit3=1，表示L-Win键

bit4=1，表示R-Shfit键

bit5=1，表示R-Ctrl键

bit6=1，表示R-Alt键

bit7=1，表示R-Win键

Byte3: Key code （请参见附录的HID Usage ID）

Byte4: 2<sup>nd</sup> code 第二个Key code （请参见附录的HID Usage ID）

例如：

若一个按键设为组合键Ctrl + Alt + Del

|        |        |        |        |
|:------:|:------:|:------:|:------:|
| Byte 1 | Byte 2 | Byte 3 | Byte 4 |
|  0x01  |  0x06  |  0x63  |  0x00  |

**c) 当Byte1为0x02时，表示该键为鼠标按键功能**

Byte2: 0x00

Byte3: Button ID

bit0=1，表示Left Key

bit1=1，表示Right Key

bit2=1，表示Middle Key

bit3=1，表示Button 4 前进

bit4=1，表示Button 5 后退

例如：

若一个按键设为中键

|        |        |        |        |
|:------:|:------:|:------:|:------:|
| Byte 1 | Byte 2 | Byte 3 | Byte 4 |
|  0x02  |  0x00  |  0x04  |  0x00  |

Byte2: 0x00

Byte3: 多媒体键值高位（请参见附录的HID Usage ID）

Byte4: 多媒体键值低位（请参见附录的HID Usage ID）

例如：

若一个按键设为被禁用

**f) 当Byte1为0x05时，表示该键为DPI功能**

Byte2: 0x00

Byte3: DPI change

00：DPI+(0 -\> 6档, 档位每次加一档，加到最高档停止，不进行循环）

01：DPI-(6 -\> 0档，档位每次减一档，减到最低档停止，不进行循环)

02: DPI+循环(0-\>6-\>0档, 档位循环加，加到最高档回到最低档，进行循环）

03: DPI-循环(6-\>0-\>6档, 档位循环减，减到最低档回到最高档，进行循环）

Byte4: 0x00

| **Error Code Definitions** |  |  |
|:---|:---|:---|
| **Err Code (Hex)** | **含义（Description）** | **触发场景说明** |
| 0x00 | 保留（Reserved） | 不应出现；若收到，视为未知状态 |
| 0x01 | 长度异常（Length Mismatch） | 包头中 len 字段与实际有效 payload 长度不符 |
| 0x02 | 参数异常（Invalid Parameter） | 模式（mode）、表ID（table_id）、DPI 值、宏ID 等错误或超出合法范围 |
| 0x03 | 命令不支持（Command Not Supported） | 收到未定义的 sub_cmd（如 0xA8、0xFF 等） |
| 0x04 | Flash 读取失败（Flash Read Error） | 从非易失存储器读取配置/宏数据时发生硬件或驱动错误 |
| 0x05 | Flash 擦除失败（Flash Erase Error） | 擦除 Flash 扇区失败（可能因写保护、电压不稳等） |
| 0x06 | Flash 写入失败（Flash Write Error） | 向 Flash 写入配置/宏数据失败 |
| 0x07 | 数据校验失败（Checksum/CRC Error） | 读取的配置块 CRC16 校验不通过，数据可能已损坏 |
| 0x08 | 表 ID 不支持（Table ID Out of Range） | 请求的子表 ID（如 table_id=3）超出设备支持范围（0~2） |
| 0x09 ~ 0xFF | 保留（Reserved for Future Use） | 未来扩展使用，请勿依赖 |
